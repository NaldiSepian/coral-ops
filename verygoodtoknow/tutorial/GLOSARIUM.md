# Glosarium
> Daftar istilah teknis dan konsep yang digunakan dalam proyek Coral-Ops.

---

### A
- **App Router**: Sistem routing baru di Next.js yang menggunakan folder `app/` untuk mendefinisikan rute dan layout.
- **API Route**: Endpoint backend yang dibuat di dalam folder `app/api/` untuk menangani request HTTP.

### B
- **Before-After Pair**: Sepasang foto (sebelum dan sesudah) yang digunakan sebagai bukti penyelesaian suatu bagian pekerjaan.
- **BWM (Best-Worst Method)**: Metode pengambilan keputusan multikriteria yang digunakan untuk menghitung bobot kriteria secara objektif.

### C
- **Client Component**: Komponen React yang dirender di browser, ditandai dengan direktif `"use client"`. Digunakan untuk interaktivitas.

### G
- **Geolocation**: Data koordinat geografis (Latitude & Longitude) yang diambil dari perangkat user.
- **GPS Check-in**: Proses verifikasi lokasi teknisi saat memulai pekerjaan untuk memastikan kehadiran di area proyek.

### I
- **Inventaris**: Daftar alat/equipment milik perusahaan yang bisa dipinjamkan ke teknisi lapangan.

### J
- **JWT (JSON Web Token)**: Metode aman untuk mentransmisikan informasi antar pihak sebagai objek JSON, digunakan untuk autentikasi.

### L
- **Leaflet**: Library open-source JavaScript untuk membuat peta interaktif yang ringan dan mobile-friendly.
- **LP Solver**: Algoritma pemrograman linear yang digunakan untuk mencari solusi optimal (bobot kriteria) dalam metode BWM.

### P
- **PostGIS**: Ekstensi database PostgreSQL yang memungkinkan penyimpanan dan query data geografis.

### R
- **RLS (Row Level Security)**: Fitur PostgreSQL (dan Supabase) untuk membatasi akses data pada level baris berdasarkan identitas user.

### S
- **Server Component**: Komponen React yang dirender di server secara default di Next.js 15.
- **SPK (Surat Perintah Kerja)**: Dokumen perintah yang berisi detail penugasan untuk teknisi.
- **Supabase**: Platform Backend-as-a-Service open source yang menyediakan database, auth, dan storage.

### W
- **Wizard Form**: Pola desain formulir yang membagi inputan panjang menjadi beberapa langkah berurutan agar lebih mudah diisi.
- **WSM (Weighted Sum Model)**: Model perhitungan skor akhir dengan mengalikan bobot kriteria dengan nilai ternormalisasi.
